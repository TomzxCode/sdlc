# Requirements: Auto Review

## Overview

An automated iterative review system that creates a loop between the original agent and a reviewer agent. After the original agent completes work, a reviewer agent examines the output, generates findings, and passes them back. The original agent then revises its work based on the findings. This continues until the reviewer finds no remaining issues or a maximum iteration count is reached.

## Stakeholders

| Stakeholder | Interest |
|---|---|
| Developers | Automated code review without manual PR review cycles |
| Teams | Enforce quality standards through automated agent review loops |

## Functional Requirements

| ID | Priority | Requirement |
|---|---|---|
| FR-01 | Must | The system shall create a review session when the user triggers an auto review on a completed session. |
| FR-02 | Must | The system shall forward session context to a reviewer agent that generates findings. |
| FR-03 | Must | The system shall pass reviewer findings back to the original agent for revision. |
| FR-04 | Must | The system shall support configurable maximum iteration count (default 15). |
| FR-05 | Must | The system shall detect when the reviewer has no remaining findings and mark the review as complete. |
| FR-06 | Should | The system shall display a banner in the chat UI showing the review status (waiting for reviewer, waiting for implementer). |
| FR-07 | Should | The system shall support a timeout for each handoff phase (default 3 minutes). |

## Non-Functional Requirements

| ID | Priority | Category | Requirement |
|---|---|---|---|
| NFR-01 | Must | Reliability | A review loop failure shall not crash the app or corrupt session state. |
| NFR-02 | Should | Performance | The review loop shall not block the user from starting new sessions. |

## Acceptance Criteria

- [ ] FR-01: Given a completed session, triggering auto review creates a linked review session.
- [ ] FR-02: Given an active review, findings are generated by the reviewer agent.
- [ ] FR-03: Given reviewer findings, the original agent receives them and revises.
- [ ] FR-05: Given no remaining findings, the review is marked complete.
- [ ] FR-06: Given an active review, a banner shows the current review status.
